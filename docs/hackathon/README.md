## What you're building

A tool that takes a property address and produces:
1. Roof measurements (total square footage, line items where possible)
2. A quote-ready estimate

Stack is up to you. We don't care if you use OpenAI, Anthropic, geometry-first, agents, or open-source — we care that it works.

## How you'll be judged

Five things matter. Top 5 advance to the live finals.

- **Accuracy** — How close your measurements come to trusted reference data, evaluated for practical accuracy and consistency across all test properties.
- **Product** — Did you bridge from measurements into a usable estimate? Would a roofer actually use the output?
- **Experience** — How does the tool feel end-to-end — address in, estimate out?
- **Craft** — Code quality, novel use of AI, engineering judgment.
- **Demo** — How you bring it to life on Saturday afternoon. Creativity. Wow factor.

---

## Tips

- **Read [`benchmark-measurements.md`](./benchmark-measurements.md) first.** Look at how EagleView and Geospan disagree — your bar isn't perfection, it's "in range."
- **Run end-to-end on one test address before optimizing.** Working pipeline > polished components.
- **Don't fabricate measurements.** Submitted numbers can be cross-checked.
- **Show your work.** A clean repo with a README beats a black-box demo.
